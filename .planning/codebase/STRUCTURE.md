## Directory Purposes

**`app/`:**
- Purpose: All runtime application code deployed to Docker
- Contains: Python modules, templates, static assets, CSV data files
- Key files: `run.py` (entry), `routes.py` (HTTP), `get_tides.py` (PDF gen), `database.py` (data), `tide_adapters.py` (APIs)
- Note: The Dockerfile explicitly copies individual files from this directory (not a blanket COPY)

**`app/templates/`:**
- Purpose: Jinja2 HTML templates rendered by Flask
- Contains: 4 templates using template inheritance from `base.html`
- Key files: `index.html` (main page with embedded JavaScript for autocomplete, form handling, country filter)

**`app/static/`:**
- Purpose: Static assets served by Flask
- Contains: CSS, favicon, example image, SEO files (robots.txt, sitemap.xml, llms.txt, ads.txt)
- Note: Root-level files (robots.txt, etc.) are served via explicit Flask routes in `routes.py`, not Flask's default `/static/` path

**`tests/`:**
- Purpose: Playwright end-to-end browser tests
- Contains: Test specs, page objects, config, utilities
- NOT deployed to Docker (excluded via `.dockerignore`)

**`scripts/`:**
- Purpose: Development and maintenance scripts run manually
- Contains: Station validation, example image updates, integration tests
- NOT deployed to Docker

**`docs/`:**
- Purpose: Project documentation and reports
- Contains: Performance benchmarks, planning docs, validation reports
- NOT deployed to Docker

**`backup/`:**
- Purpose: Timestamped backups of CSV station files
- Created by: `validate_tide_stations.py` script before modifying CSVs

## Where to Add New Code

**New Flask Route:**
- Add route handler to `app/routes.py`
- If serving a root-level static file, add both the file to `app/static/` and a route using `_serve_static_file()` helper

**New Template:**
- Create in `app/templates/`, extend `base.html` for consistent header/footer/analytics
- Reference from route handler via `render_template()`

**New API Adapter (e.g., new country's tide API):**
- Create new class in `app/tide_adapters.py` extending `TideAdapter`
- Implement `get_predictions()`, `validate_station()`, `parse_response()`
- Update `get_adapter_for_station()` factory function
- Add station import logic similar to `app/canadian_station_sync.py`

**New Database Operations:**
- Add functions to `app/database.py`
- Follow pattern: open `sqlite3.connect(DB_PATH)` context manager, parameterized queries, return dicts/lists

**New Test:**
- Add `tests/{feature-name}.spec.ts` for new test spec
- Use page objects from `tests/pages/` for interactions
- For production smoke tests, add to `tests/smoke.spec.ts`

**New Maintenance Script:**
- Add to `scripts/` directory
- These are NOT deployed (excluded by `.dockerignore`)

**New Python Module (runtime):**
- Add to `app/` directory
- IMPORTANT: Must also add explicit `COPY` line in `Dockerfile` for it to be included in the Docker image

**New Static Asset:**
- Add to `app/static/`
- If it needs a root-level URL (e.g., `/newfile.txt`), add a Flask route in `routes.py`
