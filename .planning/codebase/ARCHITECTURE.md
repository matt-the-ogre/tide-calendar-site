## Pattern Overview

**Overall:** Server-side rendered monolith with subprocess-based PDF generation pipeline

**Key Characteristics:**
- Flask web application serving HTML forms and JSON APIs
- PDF generation delegated to `get_tides.py` via `subprocess.run()` (separate Python process)
- Adapter pattern for multi-API tide data fetching (NOAA and CHS)
- SQLite database for station metadata and usage tracking
- File-based PDF caching with automatic cleanup of stale files
- Dynamic Canadian station sync from CHS API on container startup

## Data Flow

**PDF Generation Request (POST /):**

1. User submits form with station_id, year, month -> `routes.py:index()`
2. Validate inputs (station_id is numeric, year/month in range)
3. Look up place_name from database via `get_place_name_by_station_id()`
4. Construct PDF filename: `tide_calendar_{location}_{year}_{month}.pdf`
5. Check PDF cache (`os.path.exists()` + `os.path.getsize() > 0`)
6. If cached: return file via `send_file()`, set `last_place_name` cookie
7. If not cached: spawn subprocess -> `get_tides.py --station_id X --year Y --month Z`
   - `get_tides.py` calls `get_station_info()` to determine API source (NOAA or CHS)
   - `get_adapter_for_station()` factory returns `NOAAAdapter` or `CHSAdapter`
   - Adapter fetches predictions from external API with retry logic
   - Raw API response parsed into standardized CSV format
   - CSV converted to pcal date format via `convert_tide_data_to_pcal()`
   - `pcal` generates PostScript file, `ps2pdf` converts to PDF
   - Temporary files (CSV, pcal txt, PostScript) deleted
   - Station lookup logged to database (increments `lookup_count`)
8. Verify PDF exists and is non-empty
9. Return PDF via `send_file()`, clean up previous month's cached PDFs

**Quick Generate (POST /api/generate_quick):**

1. JSON request with `station_id` -> same flow as above
2. Uses `--skip_logging` flag to avoid inflating lookup counts
3. Always generates current month's calendar

**Station Search (GET /api/search_stations):**

1. Query parameter `q` (search text) and optional `country` filter
2. `search_stations_by_country()` does case-insensitive LIKE query on `place_name`
3. Results ordered by `lookup_count DESC` (popular stations first)
4. Returns JSON array of `{station_id, place_name, country, lookup_count}`

**Container Startup (run.py):**

1. `init_database()` - Create SQLite schema with migration support
2. `import_stations_from_csv()` - Import ~2100 USA stations from `tide_stations_new.csv`
3. `import_canadian_stations_from_api()` - Fetch ~73 Canadian stations from CHS API (fallback to CSV)
4. Flask app starts

**State Management:**
- Server-side: SQLite database for station metadata and lookup counts
- Client-side: Cookie `last_place_name` remembers last selected station
- Cache: File-based PDF caching in `PDF_OUTPUT_DIR`

## Error Handling

**Strategy:** Defensive error handling at each layer boundary; log errors, return user-friendly messages

**Patterns:**
- **Form validation:** `try/except ValueError` in `routes.py:index()` catches invalid year/month/station_id, renders `tide_station_not_found.html` with message
- **PDF generation failure:** After subprocess completes, checks `os.path.exists()` and `os.path.getsize() > 0`; renders error template if missing/empty
- **API adapter retry:** Both `NOAAAdapter` and `CHSAdapter` retry up to 3 times with exponential backoff (0s, 2s, 4s) on 502/503/504 errors and timeouts
- **CHS endpoint failover:** `CHSAdapter` tries two base URLs (Azure and legacy); if first fails all retries, moves to second
- **Database errors:** Caught per-function, logged, return `None` or empty list (non-fatal)
- **Station lookup logging:** `log_station_lookup()` swallows errors -- logging is not critical to PDF generation
- **Startup resilience:** Canadian station import falls back to CSV if API fails; does not crash the application
- **Custom 404:** `app/routes.py` registers `@app.errorhandler(404)` rendering `app/templates/404.html`

## Caching Strategy

**PDF File Cache:**
- Location: `PDF_OUTPUT_DIR` (env var, defaults to `app/calendars/` locally, `/data/calendars/` in production)
- Key format: `tide_calendar_{sanitized_location}_{YYYY}_{MM}.pdf`
- Cache hit: Check `os.path.exists()` and `os.path.getsize() > 0` before generating
- Cache invalidation: `cleanup_previous_month_pdfs()` deletes PDFs from prior months using regex on filename
- Cleanup trigger: Runs after every PDF serve (both cached and freshly generated)
- No TTL-based expiration; PDFs are valid for entire month

**Database as Cache:**
- Station metadata imported once at startup (threshold check: skip if >100 stations already have `place_name`)
- Canadian stations refreshed from API on every container restart

## Database Access Patterns

**Connection Pattern:** Every function opens a new `sqlite3.connect(DB_PATH)` with context manager. No connection pooling or shared connections.

**Schema:**
```sql
CREATE TABLE tide_station_ids (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    station_id TEXT UNIQUE NOT NULL,
    place_name TEXT,
    lookup_count INTEGER NOT NULL DEFAULT 1,
    last_lookup DATETIME DEFAULT CURRENT_TIMESTAMP,
    country TEXT DEFAULT 'USA',
    api_source TEXT DEFAULT 'NOAA',
    latitude REAL,
    longitude REAL,
    province TEXT
)
```

**Migration:** `init_database()` uses `PRAGMA table_info()` to check for missing columns and adds them with `ALTER TABLE`. No migration framework.

**Import Pattern:** `INSERT OR IGNORE` followed by `UPDATE` for existing records. This preserves `lookup_count` while updating metadata.

**Sync Pattern:** After import, delete stations from DB that are not in the canonical source (CSV for USA, API response for Canada).

**Query Pattern:** All queries use parameterized SQL. Search uses `LIKE` with `LOWER()` for case-insensitive matching. Results ordered by `lookup_count DESC` to surface popular stations.

## Cross-Cutting Concerns

**Logging:** Python `logging` module throughout. Configured in `run.py` with `logging.basicConfig(level=logging.INFO)`. Each adapter uses `self.logger = logging.getLogger(self.__class__.__name__)`.

**Validation:** Input validation in `routes.py:index()` -- checks station_id is numeric, year within range, month 1-12. Adapter layer validates station ID format per API.

**Authentication:** None. Application is publicly accessible with no auth.

**Configuration:** Environment variables loaded via `python-dotenv` in `run.py`. Key vars: `DB_PATH`, `PDF_OUTPUT_DIR`, `TOP_STATIONS_COUNT`, `FLASK_RUN_PORT`. Defaults provided for local development.
